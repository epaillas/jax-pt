# FFTLog Matrices

This note documents the precomputed FFTLog matrix assets used by `CLASS-PT`. `jaxpt` no longer vendors or loads these files at runtime, but the background is still useful because the current native analytic kernels follow the same general FFTLog decomposition.

## What These Files Are

- `M13oneline_N256.dat`
  - The FFTLog vector for the one-loop `P13` contribution.
- `M22oneline_N256_packed.dat`
  - The packed FFTLog matrix for the one-loop `P22` matter contribution.
- `M22basiconeline_N256_packed.dat`
  - The packed “basic” FFTLog matrix built from the core `J(ν1, ν2)` kernel. `CLASS-PT` reuses this to build the biased-tracer real-space terms.
- `IFG2oneline_N256.dat`
  - The FFTLog vector used for the `IFG2` / `bGamma3`-related real-space term.

## Why They Exist

In the `CLASS-PT` loop formulation, the cosmology-dependent part of the one-loop integrals is separated from a cosmology-independent FFTLog kernel. Once the FFTLog discretization is fixed, the expensive kernel part becomes a set of reusable matrices and vectors.

The current native port uses the same fixed discretization as the `CLASS-PT` `N256` branch:

- `N = 256`
- `k0 = 5e-5`
- `kmax = 1e2`
- FFTLog bias `b = -0.3` for matter-like terms
- FFTLog bias `b2 = -1.6000001` for the biased real-space terms

## Where They Live

These assets belong to the upstream `CLASS-PT` repository:

- `pt_matrices/matrix_gen.py`
- `pt_matrices/matrix_gen-true_original.py`
- `pt_matrices/matrix_packed.c`

The `CLASS-PT` runtime loads them from its own source tree in the `N256` branch of:

- `source/nonlinear_pt.c`

## Mathematical Form

The matrices are generated by evaluating analytic FFTLog kernels on a complex lattice of exponents

`eta_m = b + 2π i m / (N Δlogk)`

with `m` running over the FFT modes for the chosen discretization.

The generator scripts define analytic gamma-function kernels such as:

- `J(ν1, ν2)`
- `M13(ν)`
- `M22(ν1, ν2)`

and evaluate them on that lattice. The resulting complex-valued vectors or matrices are then written to disk as concatenated real and imaginary parts.

For the `M22`-type kernels, the files store the triangular part of the matrix rather than the full dense square array.

## Packing Convention

The packed `N256` `M22` files use lower-triangular storage in the same ordering expected by `CLASS-PT`. The helper

- `/Users/epaillas/code/CLASS-PT/pt_matrices/matrix_packed.c`

converts the unpacked triangular representation into the packed on-disk format used by `nonlinear_pt.c`.

## Current Status In jaxpt

The current native backend generates the needed FFTLog kernels analytically in-repo and does not ship local copies of the `CLASS-PT` matrix files.

This document remains useful as provenance for the fixed `N256` discretization and as a reminder of how the original `CLASS-PT` implementation factors the one-loop problem.
